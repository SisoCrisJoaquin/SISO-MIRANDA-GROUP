  • Location: src/ImageLoader.java
  • Purpose: Load and cache PNG images
  • Features:
    - Loads from JAR resources

    - Image caching for performance
    - Graceful error handling
    - Returns null if image not found

UPDATED: Player.java
  • Added: private BufferedImage playerImage
  • Added: private boolean useImage
  • Updated: Constructor loads player.png

UPDATED: Obstacle.java
  • Added: private BufferedImage obstacleImage
  • Added: private boolean useImage
  • Updated: Constructor loads obstacle.png
  • Updated: draw() method - displays image or fallback

UPDATED: GamePanel.java
  • Added: import java.awt.image.BufferedImage
  • Added: private BufferedImage backgroundImage
  • Added: private boolean useBackgroundImage
  • Updated: Constructor loads background.png
  • Added: drawBackgroundImage() method
  • Updated: paintComponent() to use background image

BUILD SYSTEM: run.bat
  • Checks for Java installation
  • Creates bin/ and bin/META-INF/ directories
  • Compiles all Java files: javac -d ../bin *.java
  • Creates MANIFEST.MF with Main-Class: Main
  • Builds JAR: jar cfm ../EndlessRunner.jar META-INF\MANIFEST.MF *.class

A: Run run.bat - it will tell you if Java is missing

A: Double-click play.bat (after running run.bat first)

A: YES! Put PNG files in resources/ folder and rebuild

Q: What size should my images be?
A: player.png: 30x40, obstacle.png: 40x50, background.png: 800x500+
